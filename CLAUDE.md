# Сайт — Claude Code Instructions

## Проект
Сайт для бизнеса/услуг. Это первый учебный проект — пользователь учится создавать сайты
пошагово вместе со своим учителем. Описание уточняется по ходу работы; обновляй этот
раздел, когда станет известна конкретная тематика/нишa бизнеса.

## Правила разработки (Режим 1: MVP)

### Коммуникация
- Когда задача ГОТОВА: заканчивай сообщение ✅✅✅ и "Всё, можешь тестить!"
- Во время работы: показывай % прогресса и текущий этап (например "~40% — главная страница готова")
- Общение на русском, код и переменные на английском
- Объясняй пошагово и простым языком — пользователь учится, не разработчик

### Деплой
- НИКОГДА не деплоить без явного подтверждения — всегда спрашивать "Деплоим?"

### Решения
- Если есть несколько вариантов — предложи и спроси, не выбирай сам
- Если уверенность < 90% — задай уточняющий вопрос

### Разрешения
- Пользователь дал полное разрешение на ВСЕ действия в рамках этого проекта
- НИКОГДА не спрашивать "можно ли я сделаю X" — просто делать
- Единственное исключение — деплой (git push / публикация), всегда спрашивать

## Память проекта (папка "Память")

В корне проекта есть папка `Память/` — локальное хранилище знаний о проекте (аналог
Obsidian vault, обычные markdown-файлы со связями [[имя заметки]]).

### При старте сессии
Прочитай `Память/00-home/index.md` и `Память/00-home/текущие приоритеты.md`.

### Автоматическое ведение (БЕЗ команд от пользователя)
Claude ОБЯЗАН самостоятельно, без напоминаний:
1. Сохранять заметку в `Память/sessions/` в конце каждой сессии
2. Обновлять `Память/00-home/текущие приоритеты.md` при изменении приоритетов
3. Создавать заметку в `Память/knowledge/decisions/` при принятии решений
4. Создавать заметку в `Память/knowledge/debugging/` при обнаружении багов
5. Создавать заметку в `Память/knowledge/patterns/` при выявлении паттернов
6. Создавать заметку в `Память/knowledge/integrations/` при работе с внешними API
7. Обновлять `Память/00-home/index.md` при появлении новых заметок

Пользователь НЕ должен напоминать про сохранение. Это ответственность Claude.

Правила именования заметок — УТВЕРЖДЕНИЯ, не категории:
- ✅ "Главная страница собрана на секциях Hero+Услуги+Контакты.md"
- ❌ "structure.md"

## GSD (Get Shit Done)

GSD установлен (v1.42.3, full profile, 66 скиллов) — 2026-06-16.
Claude сам использует GSD-команды когда уместно, без напоминаний пользователю.
Пользователь не должен знать названия команд.

Скиллы дизайна (оба установлены глобально):
- frontend-design — принципы уникальной вёрстки, антишаблоны, CSS-переменные
- ui-ux-pro-max — 67 стилей, 161 палитра, 57 шрифтовых пар, 99 UX-гайдлайнов

Соответствие намерений → команд (реальные установленные скиллы):
- "давай начнём проект" → /gsd-new-project
- "давай обсудим этот модуль" → /gsd-discuss-phase
- "делай" / "погнали" → /gsd-execute-phase
- "проверь" → /gsd-verify-work
- "что дальше?" / "как дела?" → /gsd-progress (не gsd-next — такого нет)
- мелкая задача → /gsd-quick или /gsd-fast
- идея/набросок → /gsd-explore или /gsd-sketch
- отладка → /gsd-debug

## Режим 2: Продакшен (включить когда появятся реальные пользователи)

### Перед ЛЮБЫМИ изменениями кода
1. Анализ — объясни как работает текущий код
2. Зависимости — что затрагивает изменение
3. План — пошаговый, до написания кода

### Правила изменений
- Минимальный Diff — менять только необходимое
- НИКОГДА не переписывать целые файлы без просьбы
- НИКОГДА не удалять существующий код без просьбы
- НИКОГДА не рефакторить рабочий код без просьбы
- Один модуль за раз, маленькие изменения
- Итерации: изменение → проверка → следующее

### Приоритет: Стабильность > Совместимость > Минимум изменений > Новый функционал

## Профиль пользователя

- Учится создавать сайты с нуля вместе со своим учителем, это первый проект
- Не разработчик — говорит на обычном языке, не на техническом
- Не хочет учить команды — Claude сам разбирается
- Не хочет подтверждать действия — полное разрешение на всё (кроме деплоя)
- Ценит скорость и простоту, общение на русском
